## What this plugin provides

The `@lando/wordpress` plugin adds a `wordpress` recipe to Lando — a single configuration block that sets up everything you need for local WordPress development:

- **PHP 5.3–8.5+** — Pin the exact PHP version WordPress requires
- **Apache or Nginx** — Switch web servers with one config line
- **MySQL, MariaDB, or PostgreSQL** — Choose your database backend
- **WP-CLI** — Run `lando wp` commands directly in your environment
- **Composer** — Manage PHP dependencies without installing anything locally
- **Xdebug** — Enable step debugging with a single config toggle
- **Database import/export** — `lando db-import` and `lando db-export` built in

## Quick example

Add a `wordpress` recipe to your Landofile to get started:

```yaml .lando.yml
name: my-wordpress-site
recipe: wordpress
config:
  php: '8.3'
  database: mysql:8.0
```

Then run `lando start` and your local WordPress environment is live.

<Steps>
  <Step title="Install Lando">
    [Download and install Lando](https://docs.lando.dev/getting-started/installation.html) for your platform. The WordPress plugin is included by default.
  </Step>
  <Step title="Initialize your project">
    Run `lando init` in your project directory to generate a Landofile, or create one manually with the `wordpress` recipe.
  </Step>
  <Step title="Start the environment">
    Run `lando start` to build and launch your WordPress containers. Lando prints the local URL when ready.
  </Step>
  <Step title="Configure WordPress">
    Use `lando wp config create` to set up `wp-config.php` with your local database credentials, then install WordPress with `lando wp core install`.
  </Step>
</Steps>
